26  /**
27   * @author park
28   * <p> An <code>ICGConcept</code> is just an ordinary topic
29   *  in a topic map, except that it has in and out edges.</p>
30   * <p> Since it is a topic in a topic map, it might end up in
31   *  more than one graph <em>context</em>, so we can use the
32   *  {@link ICGConceptLinkStruct} structure to model each in and each out
33   *  link, where that struct's <em>contextLocator</em> is the locator
34   *  associated in which Conceptual graph which those links are valid.
35   *  Each struct is associated with a linkType as well.
36   *  </p>
37   *  <p> An <code>ICGConcept</code> is a <em>topic</em> which is a
38   *  <em>casting (role)</em> of a specific subject in the topicmap</p>
39   *  <p>It will be modeled as a specific type of <em>Pivot</em></p>
40   */
